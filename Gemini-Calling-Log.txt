### What to Look For

Before you start, know what to search for. When combing through logs, you're typically looking for keywords that indicate a problem. Common keywords include:

*   `error`
*   `failed` / `failure`
*   `critical`
*   `warning`
*   `denied`
*   `timeout`
*   `segfault` (segmentation fault)
*   `panic`

---

### For Linux (e.g., Ubuntu, CentOS, Fedora, Arch)

Most modern Linux systems use `systemd`, which has a powerful logging tool called `journalctl`. This is the best place to start.

#### Method 1: Using `journalctl` (Recommended)

Open your terminal and use the following commands. You may need to run them with `sudo`.

1.  **View All Errors Since Last Boot:**
    This is the most useful command to start with. It shows all logs with a priority of "error" (level 3) or higher.
    ```bash
    journalctl -p 3 -b
    ```
    *   `-p 3` filters for messages of priority "error" and higher (2=critical, 1=alert, 0=emergency).
    *   `-b` shows messages only from the current boot.

2.  **Follow Logs in Real-Time:**
    To see new errors as they happen.
    ```bash
    journalctl -f -p 3
    ```

3.  **Check Logs for a Specific Service:**
    If you suspect a particular program (e.g., `nginx` web server, `sshd` for SSH) is the problem.
    ```bash
    journalctl -u nginx.service -b
    ```

4.  **Check Logs from a Specific Time:**
    To see errors that occurred in the last 15 minutes.
    ```bash
    journalctl -p 3 --since "15 minutes ago"
    ```

#### Method 2: Checking Traditional Log Files

For older systems or for specific logs not covered by `journald`, you can look in the `/var/log` directory.

1.  **Search the System Log:**
    ```bash
    grep -i "error" /var/log/syslog
    ```
    *(On Red Hat/CentOS, use `/var/log/messages`)*

2.  **Check Kernel/Driver Messages:**
    This is great for hardware, USB, or driver issues.
    ```bash
    dmesg | grep -i "error"
    ```

### For Windows (10, 11, Server)

Windows uses a graphical tool called **Event Viewer** which is the primary way to check logs.

#### Method 1: Using Event Viewer (GUI)

1.  Press **Win + R** to open the Run dialog.
2.  Type `eventvwr.msc` and press Enter.
3.  In the left-hand pane, expand **"Windows Logs"**.
    *   **Application:** Logs from software and applications. Check here for program crashes.
    *   **System:** Logs from Windows system components, services, and drivers. Check here for OS-level errors, startup problems, or hardware issues.
4.  Click on either **Application** or **System**. In the right-hand "Actions" pane, click **"Filter Current Log..."**.
5.  In the filter dialog, check the boxes for **"Critical"** and **"Error"**. Then click **OK**.

This will show you a clean list of only the critical errors and error events, making it much easier to spot problems.

#### Method 2: Using PowerShell (Command Line)

For a quicker, command-line approach, you can use PowerShell.

1.  Open PowerShell as an Administrator.
2.  **Get Application Errors from the last 24 hours:**
    ```powershell
    Get-WinEvent -FilterHashtable @{LogName='Application'; Level=2; StartTime=(Get-Date).AddDays(-1)}
    ```

3.  **Get System Errors from the last 30 minutes:**
    ```powershell
    Get-WinEvent -FilterHashtable @{LogName='System'; Level=2; StartTime=(Get-Date).AddMinutes(-30)}
    ```
    *   `Level=2` corresponds to "Error". `Level=1` is "Critical".

---

### For macOS

macOS has a modern, unified logging system accessible via the **Console** app or the `log` command.

#### Method 1: Using the Console App (GUI)

1.  Open Spotlight (**Cmd + Space**).
2.  Type `Console` and press Enter.
3.  In the Console app sidebar, click on **"Errors and Faults"**.
4.  This view automatically filters for the most severe log messages across the entire system.
5.  You can use the search bar at the top-right to filter for specific applications or processes (e.g., `backupd` for Time Machine issues).

#### Method 2: Using the `log` command (Terminal)

1.  Open the Terminal app.
2.  **Stream Errors in Real-Time:**
    This is very useful for watching what happens when you perform an action that causes an error.
    ```bash
    log stream --level error
    ```

3.  **Show Errors from the Last Hour:**
    ```bash
    log show --predicate 'eventType == "error"' --last 1h
    ```

### Next Steps

Once you find an error message, **copy the full message**. If you don't understand it, paste it here or into a search engine. The context around the error (what you were doing, what time it happened) is also very important for diagnosis.
